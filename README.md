# Микшер громкости для macOS

Menu bar приложение, которое даёт то, чего нет в macOS из коробки: **отдельный
ползунок громкости для каждого приложения** — как микшер в Windows.

- Ползунок 0–100% и mute для каждого приложения, играющего звук
- VU-метр: видно, кто именно шумит
- Мастер-ползунок общей громкости
- Настройки запоминаются по bundle ID: приглушил Telegram один раз — он
  останется приглушённым и после перезапуска
- Никаких драйверов: официальный API CoreAudio process taps (macOS 14.4+)

## Как это работает

На каждое играющее приложение создаётся muted process tap — система глушит
оригинальный вывод процесса и отдаёт его аудиопоток нам. Поток проходит через
приватное агрегатное устройство: сэмплы умножаются на коэффициент громкости
(перцептивная квадратичная кривая) и уходят в текущее устройство вывода. Там же
считается RMS для VU-метра. Смена устройства вывода пересоздаёт цепочку
автоматически.

## Сборка и запуск

Нужен только Swift toolchain из Command Line Tools (Xcode не требуется), macOS 15+.

```bash
./build.sh                          # соберёт build/Микшер громкости.app
open "build/Микшер громкости.app"
```

Тесты (обёртка добавляет пути к Testing.framework из CLT):

```bash
./test.sh
```

При первом запуске macOS может спросить разрешение на **запись системного
звука** — без него регулировать громкость приложений нельзя. Если отказали,
панель покажет кнопку с переходом в нужный раздел настроек.

## Ограничения v1

- Роутинг приложений на разные устройства вывода — нет (все играют в текущее)
- Эквалайзера и горячих клавиш нет
- Хелперы браузеров показываются по родительскому приложению (Chrome — как
  Chrome, а не «Chrome Helper»)
- Подпись ad-hoc: сборка для личного использования, не для раздачи

## Структура

- `Sources/VolumeMixerCore` — логика: монитор аудиопроцессов, tap-контроллеры,
  движок, настройки, системная громкость
- `Sources/VolumeMixerApp` — SwiftUI: MenuBarExtra-панель, строки приложений,
  онбординг разрешения
- `docs/superpowers/` — дизайн и план реализации
