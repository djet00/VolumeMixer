# Микшер громкости для macOS — дизайн

Дата: 2026-07-15
Статус: утверждён (обсуждение в чате)

## Цель

Menu bar приложение для macOS, дающее то, чего нет в системе: регулировку громкости
каждого приложения по отдельности — как микшер в Windows.

## Окружение и ограничения

- macOS 26.5 (минимальная поддерживаемая версия — 14.4, где появился API process taps).
- Swift 6.3, только Command Line Tools (без Xcode) — сборка через Swift Package Manager,
  .app-бандл собирается скриптом, подпись ad-hoc.
- Никаких виртуальных аудиодрайверов и установки чего-либо в систему.

## Функции первой версии

1. Список приложений, которые сейчас воспроизводят звук: иконка, имя, ползунок 0–100%,
   кнопка mute.
2. VU-метр — живой индикатор уровня сигнала у каждого приложения.
3. Мастер-ползунок общей громкости системы внизу панели.
4. Запоминание настроек: громкость/mute по bundle ID сохраняются в UserDefaults и
   применяются при следующем появлении приложения в списке.
5. Онбординг разрешения: если захват системного звука не разрешён, панель объясняет
   зачем нужно разрешение и ведёт в Системные настройки.

## Технический подход

Официальный API CoreAudio process taps (macOS 14.4+):

- Список аудиопроцессов система отдаёт через `kAudioHardwarePropertyProcessObjectList`;
  кто реально играет — по `kAudioProcessPropertyIsRunningOutput` (+ подписка на
  изменения, чтобы список в панели обновлялся сам).
- На каждый играющий процесс создаётся muted-tap (`CATapDescription` с mute,
  `AudioHardwareCreateProcessTap`): система глушит оригинальный вывод процесса,
  а его аудиопоток отдаёт нам.
- Tap включается в приватное агрегатное устройство вместе с текущим устройством
  вывода. В IOProc агрегата: читаем сэмплы tap'а → умножаем на громкость приложения
  (mute = 0) → пишем в выход. Там же считаем RMS для VU-метра.
- Громкость — линейный множитель с перцептивной кривой (квадрат позиции ползунка),
  чтобы ползунок ощущался равномерно.
- Мастер-громкость — обычная громкость устройства вывода по умолчанию
  (`kAudioDevicePropertyVolumeScalar`), без перехвата.
- Смена устройства вывода по умолчанию → пересоздание агрегатов (подписка на
  `kAudioHardwarePropertyDefaultOutputDevice`).

## Архитектура

SPM executable, один процесс, LSUIElement (без иконки в Dock). Модули:

| Модуль | Ответственность | Зависимости |
|---|---|---|
| `AudioProcessMonitor` | Следит за списком аудиопроцессов CoreAudio, маппит PID → приложение (имя, bundle ID, иконка через NSRunningApplication) | CoreAudio, AppKit |
| `ProcessTapController` | Один экземпляр на приложение: tap + агрегат + IOProc, атомарный gain, публикация уровня VU | CoreAudio |
| `AudioEngine` | Оркестратор: monitor → создать/убрать контроллеры, применить сохранённые громкости, пересоздать при смене устройства вывода | оба выше, SettingsStore |
| `SettingsStore` | Персистентность bundleID → (volume, muted) в UserDefaults | Foundation |
| `SystemVolume` | Чтение/запись громкости устройства вывода по умолчанию | CoreAudio |
| `UI` | NSStatusItem + SwiftUI-панель: строки приложений (иконка, ползунок, mute, VU), мастер-ползунок, экран онбординга разрешения | SwiftUI, AppKit |

Поток данных: UI меняет громкость → AudioEngine → атомарная запись gain в контроллер
(читается real-time потоком без блокировок) → параллельно SettingsStore сохраняет.
VU: IOProc пишет уровень в атомик → UI опрашивает таймером ~30 Гц пока панель открыта
(закрыта — не опрашивает, ноль нагрузки).

## Права и упаковка

- Info.plist: `NSAudioCaptureUsageDescription`, `LSUIElement = true`.
- Первый запуск: macOS показывает системный запрос на запись системного звука.
  Отказ → экран онбординга с кнопкой в Системные настройки → Конфиденциальность.
- Скрипт `build.sh`: `swift build -c release` → сборка `Микшер громкости.app`
  (bundle + Info.plist + иконка) → `codesign` ad-hoc.

## Обработка ошибок

- Нет разрешения — онбординг вместо списка.
- Ошибка создания tap для конкретного процесса — приложение показывается без
  ползунка, с пометкой; остальные работают.
- Процесс умер — контроллер и строка убираются (подписка на список процессов).
- Смена/отключение устройства вывода — тихое пересоздание цепочки.

## Тестирование

- Unit (swift test): SettingsStore, перцептивная кривая громкости, логика
  diff'а списка процессов в AudioEngine.
- Интеграционно вручную: воспроизведение из 2–3 приложений одновременно,
  проверка независимой громкости, mute, VU, персистентности после перезапуска,
  смены наушники/динамики.
